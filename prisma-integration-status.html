Goal
Automated Two-Way ERP Integration

Provide a reliable, automated two-way ERP integration between Basis and Prisma that programmatically synchronizes campaign planning, IO management, placement-level delivery data, and billing. Replaces manual entry, spreadsheets, and disconnected billing workflows.

Unlocks HoldCo agency relationships -- OMD, GroupeConnect, Zimmerman -- with billions in GDMS. Eliminates manual reconciliation required for Prisma-originated campaigns in billing and finance workflows. Baseline to be established at pilot onboarding; target improvement metrics TBD post-pilot.

Work Breakdown

Six buckets with moderate sequential dependencies. Updated post-Chicago onsite, May 2026.

Scope Note
The work has been reorganized from 5 projects to 6 buckets following the Chicago onsite. This breakdown is still evolving -- definition and boundaries may change as solution detail is finalized. Build authorization is not yet granted; spec and design work continues.
1
Authorization and Configuration
OpenID Connect + API key model. Basis user authenticates via Prisma OAuth flow; org context established per session. Email matching required short-term; medium-term path to OIDC agreed. Get-orgs endpoint coming in June 3 release.
HIGH confidence Blocked -- Jun 3
2
Master Data Linking
Map Basis entities (vendors, properties, sites) to Prisma's supplier/provider/site hierarchy. Site/supplier mapping: send Basis name on create, copy GET response on update. Agency-specific configurations supported (e.g. integrated providers like Meta may be required or optional).
HIGH confidence
3
Campaign Shell Management
Map Basis campaigns to Prisma media plans. Basis sends campaign details on create; copies Prisma data on update. Prisma owns IO creation. Supports planning-led workflow where agency plans in Basis and pushes to Prisma media plans.
HIGH confidence
4
Media Plan Line Item Management
Outbound: user-initiated push of placements from Basis to Prisma (requires user identity for audit trail). Inbound: receive Prisma placement revision notifications, fetch updated data, apply changes to Basis line items (involves LIRA). Revisions keeping total IO cost unchanged stay within Prisma. Cost changes routed back to Basis via difference-resolution workflow. Start with manual push; add automation based on usage.
LOWER confidence Most complex
5
Delivery Data
Push Basis delivery data to Prisma using Send Supplier Delivery API. MVP requires both automated (overnight) and manual (user-triggered) options. DSP tactic add-ons require two-step push. Show last data synced date/time. Note: current Prisma API provides monthly rollups only -- daily granularity (pacing) remains an open post-PoC gap.
MEDIUM confidence Pacing gap open
6
Notifications / Vendor IO and Campaign Approval Sync
Subscribe to and handle Prisma webhook/notification events (campaign updates, placement changes, delivery updates). Notifications carry minimal payload -- must fetch full data on receipt. Feeds into buckets 3 and 4. Vendor IO and campaign approval sync included; IO status scope (all direct vendors vs. eSeller only) to be confirmed with Prisma.
MEDIUM confidence
Out of Scope -- V1
Monthly billing breakouts/distributions · Bulk campaign auto-matching · Automated Prisma→Basis revision sync (start manual) · DSP integration (deferred from V1) · Daily delivery granularity (monthly totals only in V1)
